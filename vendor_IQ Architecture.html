Cluster A — Identity, Financials & Leadership (Phase 1 · Runs first)
Identity & Financial Metrics Agent
Resolves ticker, SEC CIK, HQ, public/private status, revenue, ARR, EBITDA, P/E, market cap. SEC 10-K / 10-Q as primary evidence.
Executive Leadership Agent
CEO, CTO, CFO, board — credentials, tenure, LinkedIn, LinkedIn connections to customers / investors.
Market Position Agent
TAM/SAM/SOM, analyst quadrant positioning, market share signals, growth trajectory.
SEC Filing Scan Agent
Risk factors, revenue segments, MD&A, litigation, guidance. Requires public ticker or CIK. Cites filing date + form type.
Output contract: Section-ready Markdown only · raw evidence URLs inline · "Not found in public sources reviewed" for gaps · no HTML/CSS/JSON artifacts
✕ close
Cluster B — Business Model, Products & AI/ML (Phase 2 · After identity is known)
Business & Revenue Model Agent
SaaS, usage-based, marketplace, perpetual, channel — pricing model, revenue mix, cohort retention where available.
Product / Service Catalog Agent
All named products, editions, bundles, SKUs — target buyers, integrations, certifications, end-of-life signals.
Product-to-Capability Map Agent
Maps each product to functional capability clusters. Produces structured table for cross-product comparison.
AI / ML / GenAI Capabilities Agent
LLM integrations, proprietary models, fine-tuning, RAG, copilot features, AI roadmap signals per product.
Collaboration: Cluster B feeds product names to Clusters C, D, and E. Orchestrator waits for catalog completeness before dispatching architecture agents.
✕ close
Cluster C — Architecture & Technical Due Diligence (Phase 3 · Runs parallel to D)
Product Architecture Agent
Per-product: cloud-native / on-prem / hybrid, microservices, monolith, data flow, API surface from official docs.
Information Architecture Agent
Data models, storage, retention, data residency, multi-tenancy isolation, classification.
Application Architecture Agent
Frontend stack, backend stack, SDK/CLI, APIs (REST/GraphQL/gRPC), event-driven patterns.
Security Architecture Agent
Certifications (SOC2, ISO27001, FedRAMP), RBAC, encryption in transit/at rest, pen-test posture.
Standards & Interoperability Agent
Open standards adoption, connector ecosystem, iPaaS, webhook support, data portability.
Resiliency & DR Agent
SLA commitments, RTO/RPO, multi-region, failover, incident history from status pages.
Collaboration: References product names from Cluster B. Feeds security posture and compliance findings to Cluster E (risk / competitive).
✕ close
Cluster D — Ecosystem, Customers & Public Sector (Phase 3 · Runs parallel to C)
Customer Segments Agent
Named customer logos vs inferred segments — case studies, G2/Gartner reviews, public PR mentions only.
Suppliers, Cloud & Partners Agent
AWS/Azure/GCP dependency, ISV partners, resellers, OEM relationships, ecosystem lock-in risk.
Gov Contracts & Public Sector Agent
SAM.gov awards, SBIR, FedRAMP, ATO, marketplace listings — separates direct awards from partner-led resale.
Milestones & Acquisitions Agent
Funding rounds, M&A history, major product launches, analyst recognitions, partnership announcements.
Case Studies & Benefits Agent
Measurable ROI claims, named clients, before/after metrics from official vendor case study pages.
Collaboration: Customer signals feed Cluster E's competitive Pugh Matrix weighting. Public sector findings cross-referenced with Cluster C security certifications.
✕ close
Cluster E — Competitive, Risk & Market Perception (Phase 4 · Runs after product + market evidence exists)
ESG, Sustainability & Privacy Agent
Carbon goals, DEI commitments, GDPR/CCPA posture, responsible AI disclosures, trust center review.
Analyst & Market Sentiment Agent
Gartner, Forrester, IDC, G2, Capterra, Reddit/HN signals — tone, recency, outlier opinions.
Cybersecurity & Litigation Agent
CVE history, breach disclosures, CISA advisories, SEC litigation flag, regulatory enforcement actions.
Forward-Looking Indicators Agent
Product roadmap leaks, job postings analysis, patent filings, conference keynote themes.
Product Moat & Differentiation Agent
Network effects, switching costs, data flywheel, patents, ecosystem lock-in — per named product.
Competitive Pugh Matrix Agent
Weighted scoring matrix: vendor vs named competitors across capability dimensions. Sortable HTML table.
Strengths, Gaps & Risks Agent
SWOT synthesis drawing evidence from all upstream clusters. Flags unresolved contradictions.
Collaboration: Draws from ALL upstream clusters. Conflict detection here surfaces back to orchestrator for resolution before Cluster F.
✕ close
Cluster F — Finalization & Synthesis (Phase 5 · Runs last, after all sections collected)
Reference Normalization Agent
Extracts raw URLs from all Markdown sections · deduplicates · assigns [1], [2], [3]... citations · inserts Reference Links section before Quality JSON.
Quality Evaluation Agent
Runs all Quality Gate checks · produces Quality Document JSON as the final visible report section · section-level confidence scoring.
Markdown → HTML Conversion Agent
Produces self-contained exhaustive_final_report.html: sticky header, clickable TOC, sortable tables, collapsible sections, clickable citations, print CSS — zero CDN dependencies.
Single output rule: Only one persistent artifact produced — artifacts/exhaustive_final_report.html. All intermediate Markdown is internal representation only.

Quality Gate — Pre-Conversion Evaluation
All checks must pass before HTML conversion is triggered. Failed checks block conversion; partial Markdown with fallback markers is preferred over hallucinated content.
✓
Section completeness
All 29 required sections present. Missing sections must contain explicit fallback block, not blank or omitted.
✓
Citation coverage
Every factual claim cited with [n] or marked "Not found in public sources reviewed". Zero uncited assertions.
✓
Reference link integrity
Every raw URL resolved to a numbered reference. Reference Links section appears immediately before Quality Document JSON.
✓
Table validity
All Markdown tables have correct column alignment, no broken pipes. Pugh Matrix must exist when competitive analysis is attempted.
!
Financial visual presence
Public-company financial charts (revenue, EBITDA, ARR trends) must be represented when ticker / SEC CIK is known.
✗
No unsupported claims
Zero invented customers, certifications, architecture details, SEC specifics, or executive credentials. Hard block on hallucination.
✗
No rogue sub-agent artifacts
No sub-agent output includes HTML, CSS, JavaScript, separate JSON artifacts, or intermediate file instructions. Single-output rule enforced.
✓
Quality Document JSON
Final visible section must be Quality JSON: section-level confidence scores (high/medium/low), unresolved gaps, evidence limitations, overall quality score.
